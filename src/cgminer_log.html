Field reference
Field Notes
STATUS block
STATUS.STATUSResult class: S (success), I (info), W (warning), E (error), F (fatal).
STATUS.MsgRecord type, for example Summary or CGMiner stats.
STATUS.CodeNumeric status code for the response type.
STATUS.WhenUnix timestamp when this status record was produced.
STATUS.Descriptioncgminer version/build string.
SUMMARY block - throughput, shares, and jobs
SUMMARY.ElapsedSeconds since miner start.
SUMMARY.MHS avAverage hashrate in MH/s.
SUMMARY.MHS 5sShort-window hashrate in MH/s.
SUMMARY.MHS 1m / 5m / 15mRolling hashrate windows in MH/s.
SUMMARY.Total MHTotal accumulated work in MH units.
SUMMARY.AcceptedTotal accepted shares.
SUMMARY.RejectedTotal rejected non-stale shares.
SUMMARY.StaleTotal stale shares.
SUMMARY.Found BlocksFound block counter (often 0 on home units).
SUMMARY.Best ShareHighest accepted share difficulty.
SUMMARY.GetworksTotal jobs/work items received.
SUMMARY.Network BlocksNetwork block events seen by the miner.
SUMMARY.UtilitySubmitted shares per minute from cgminer.
SUMMARY.Work UtilityReceived jobs per minute.
SUMMARY.Hardware ErrorsTotal hardware/CRC/hash errors.
SUMMARY.Difficulty AcceptedTotal accepted share difficulty.
SUMMARY.Difficulty RejectedTotal rejected non-stale share difficulty.
SUMMARY.Difficulty StaleTotal stale share difficulty.
SUMMARY.Device Hardware%Device hardware error percentage.
SUMMARY.Device Rejected%Device rejected share percentage.
SUMMARY.Pool Rejected%Pool rejected share percentage.
SUMMARY.Pool Stale%Pool stale share percentage.
SUMMARY.Get FailuresFailed work fetch attempts.
SUMMARY.Remote FailuresRemote communication/work failures.
SUMMARY.Local WorkLocal work counter (often 0 on Nano3).
SUMMARY.DiscardedDiscarded work counter (often 0).
SUMMARY.Last getworkOriginal cgminer meaning: Unix time of last job; Nano3 firmware commonly reports 0.
STATS block - top-level keys
STATS.STATSSTATS record index (commonly 0).
STATS.IDDevice identifier (for example AVANANO0).
STATS.ElapsedElapsed seconds in stats context.
STATS.CallsDriver/internal call counter (firmware-dependent).
STATS.Wait / Max / MinDriver timing/debug values; interpretation is firmware/vendor specific.
STATS.MM CountMM record count in STATS payload.
STATS.Nonce MaskActive nonce-mask setting reported by firmware.
STATS.MM ID0Packed telemetry string; all bracket tokens below are parsed from this value.
MM ID0 - identity and control
MM ID0: Ver / HVerFirmware and hardware version identifiers.
MM ID0: DNA / HDNADevice and hashboard identifiers.
MM ID0: CoreASIC core family/model identifier.
MM ID0: LedLED state/status value.
MM ID0: FreqEquivalent frequency telemetry value reported by firmware.
MM ID0: WORKLEVELWork mode value (0 Eco, 1 Normal, 2 Super).
MM ID0: MPOConfigured power setpoint (W).
MM ID0: ADJAdjustment state value used by firmware tuning logic.
MM ID0: SoftOFFSoft-off state marker.
MM ID0: VoVendor-specific board telemetry value.
MM ID0: TAASIC chip count.
MM ID0 - network and failover
MM ID0: PINGPool/network latency in milliseconds.
MM ID0: NETFAIL[...]Disconnect/reconnect timestamp buffer; on Nano3 samples these values behave like Unix epoch timestamps, and 0 means an empty slot.
MM ID0: PFCntProtection/power-fail style counter used by firmware diagnostics.
MM ID0: ECHU / ECMMError counters for control-unit and module communication paths.
MM ID0 - thermals and fan
MM ID0: Temp / OTempAirflow in and airflow out temperatures (degC).
MM ID0: TAvg / TMax / TarTASIC average, ASIC max, and ASIC target temperatures (degC).
MM ID0: MTavg / MTmaxAdditional ASIC average/max telemetry values.
MM ID0: Fan1 / FanRFan RPM and fan duty percentage.
MM ID0: PVT_T0[...]Per-chip temperature telemetry array.
MM ID0 - hashrate, work, and errors
MM ID0: LWLocal work counter from the controller/MM to the chips.
MM ID0: GHSspd / GHSmm / GHSavg / MGHSCurrent, reference/mid-window, average, and related hashrate telemetry values in the GH/s family.
MM ID0: WUWork utility telemetry value, aligned with work/share rate reporting.
MM ID0: MH / HW / DHW / DH / DHspdHardware error counters and related error-rate telemetry used for error metrics.
MM ID0: MW0[...]Per-chip work/nonces distribution array.
MM ID0: CRCCommunication CRC error counter.
MM ID0 - power and tuning arrays
MM ID0: PS[...]Power/status tuple. In this UI, the last PS value is treated as actual power.
MM ID0: ATA0 / ATA1 / ATA2[...]Per-profile tuning tuples; the first values are used as power markers in this UI.
MM ID0: PLL0[...]Core-count distribution across the configured frequency buckets.
MM ID0: SF0[...]Frequency bucket values in MHz that correspond to the PLL0 distribution.
MM ID0: PVT_V0[...]Per-chip voltage telemetry array reported as raw firmware values.
